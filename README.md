# Feedback Web App – QA Manual Test Plan

## 📌 Project Overview
This repository contains the **Quality Assurance (QA) testing documentation** for the **Feedback Web Application**.  
The goal of this project is to ensure that the application's core functionalities work correctly and consistently across different environments and browsers.

The testing process includes functional validation, UI verification, API testing, and regression testing to maintain system stability and performance.

# 🧪 Types of Testing Performed

| Testing Type | Description |
|--------------|------------|
| Functional Testing | Verifies that all features work according to requirements |
| UI Testing | Ensures interface elements display and behave correctly |
| Browser Compatibility Testing | Confirms the application works across different browsers |
| API Testing | Validates backend API responses and integrations |
| Automation Testing | Uses automated scripts to execute repeated tests |
| Regression Testing | Ensures new updates do not break existing functionality |
