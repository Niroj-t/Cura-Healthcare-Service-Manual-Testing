# Bug List Documentation

## 📌 Overview

This folder contains the **bug tracking documentation** identified during the manual testing of the **Feedback Web Application**.

Bugs were discovered while executing different **test scenarios and test cases** during the QA testing process. Each bug entry references the corresponding **Scenario TID** and **Test Case ID** to maintain traceability between the defect and the test case where the issue was identified.

---

## 🐞 Bug Report Fields

| Field           | Description                                                           |
| --------------- | --------------------------------------------------------------------- |
| Scenario TID    | Unique identifier of the test scenario where the issue was discovered |
| Test Case Id    | The specific test case that triggered the bug                         |
| Bug Description | Detailed explanation of the defect found in the system                |
| Status          | Current state of the bug (Open, Closed, In Progress)                  |
| Comments        | Additional notes, observations, or developer feedback                 |

## 🎯 Purpose

The bug list helps to:

* Track defects discovered during testing
* Maintain traceability between test cases and defects
* Provide clear documentation for developers
* Monitor the progress of bug fixes
* Improve overall software quality

Proper bug documentation ensures that all identified issues are recorded and resolved efficiently before deployment.
