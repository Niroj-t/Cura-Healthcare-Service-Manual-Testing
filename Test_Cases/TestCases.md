# Test Cases Documentation

## 📌 Overview

This folder contains the **manual test cases** created for the **Feedback Web Application**.
The test cases verify that the application's core functionalities work according to the defined requirements.

The test cases cover different modules of the application and include detailed steps, expected results, and execution status.

---

## 🧩 Modules Covered

| Module    | Description                                        |
| --------- | -------------------------------------------------- |
| Login     | Verifies user authentication functionality         |
| Dashboard | Ensures correct display of user data and interface |
| Message   | Tests feedback message submission and display      |
| Logout    | Confirms users can securely log out of the system  |

---

## 📊 Test Case Details

| Field           | Description                                    |
| --------------- | ---------------------------------------------- |
| Test Case ID    | Unique identifier for each test case           |
| Module          | Application module being tested                |
| Test Scenario   | High-level description of what is being tested |
| Test Steps      | Step-by-step instructions to execute the test  |
| Expected Result | Expected system behavior                       |
| Actual Result   | Actual system behavior after execution         |
| Status          | Pass / Fail / Blocked                          |

## 🎯 Purpose

The purpose of these test cases is to:

* Validate system functionality
* Identify defects in the application
* Ensure application stability
* Maintain consistent behavior across different browsers and environments

These test cases help ensure that the **Feedback Web App functions correctly and meets quality standards.**
